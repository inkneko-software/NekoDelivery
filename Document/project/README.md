## NekoDelivery - 墨猫速递 的项目说明文档
本文档描述了与`墨猫速递`项目相关的细节

### 功能模型

 - 角色划分
  - 用户 
    - 作为快递代取需求的发送方
    - 可申请成为配送者
  - 配送者
    - 作为代取需求的解决方, 进行快递配送
  - 快递站
    - 验证配送者与其拿取的快递是否匹配
  - 客服
    - 作为配送者需求的验证方
    - 纠纷的解决
 - 业务逻辑划分
	- 用户层
	  - 向系统中添加快递数据
	  - 向系统中添加代取需求
	  - 与配送层的交流
	  - 与客服的交流
	- 配送层
	  - 解决系统中快递的代取需求
	  - 与用户层的交流
	  - 与客服的交流
	- 快递站层
	  - 对配送者与快递物品匹配关系的验证
	- 客服层
	  - 对用户成为配送者的过程进行身份验证

### 功能细节

- 用户
  - 注册/登录/注销/修改密码
  - 快递数据的添加/查询/更新/删除
  - 快递代取需求的发布/查询/更新/删除
  - 快递的收取确认
  - 与配送者的即时通讯交流
  - 提交身份证+学生证信息, 申请成为配送者
  - 与客服进行联系
- 配送者
  - 快递代取需求的查询/接单
  - 当前需配送的快递需求查询/取消
  - 快递的配送确认
  - 与用户的即时通讯交流
  - 与客服进行联系
- 配送站
  - 验证配送者是否为其所持快递的代取者
  - 快递出站的记录
- 客服
  - 验证成为配送者申请, 检查申请信息[]
  - 解决用户层发起的纠纷解决, 如果丢件, 提供该代取者的身份信息
  - 解决配送者发起的纠纷解决

### 项目技术架构

- 前端
  - 安卓端
    - 使用Java进行安卓客户端的编写, 承载了用户/配送者/快递站的业务展示/业务操作.
  - 浏览器端
    - 使用HTML + JavaScript进行编写, 承载了客服的业务操作.
- 后端
  - 服务器端
    - 业务逻辑层使用C++进行实现
    - 测试使用googleTest框架
  - 数据持久化
    - 业务数据使用MySQL进行持久化
    - 日志数据保存至硬盘的文件中
